Overview

IDMxPPM neo-Seoul is a desktop application for authoring Information Delivery Manuals (IDMs) compliant with ISO 29481-1 and ISO 29481-3. Built for the AEC industry, it provides an integrated environment to model use cases, define exchange requirements, create BPMN 2.0 process diagrams, and export machine-readable IDM specifications.

The startup screen presents pathways into the application

When you launch the application, the startup screen offers these options:

Startup Options

1
Blank Project — Start a brand-new IDM specification from scratch with an empty BPMN canvas and default metadata.
2
Sample Project — Open a pre-built example IDM ("Generic Data Exchange") to explore the application's features with realistic data already populated.
3
Open File — Load an existing .idm, .xml, .xppm, .zip, or .bpmn project file from your computer.
4
Open from Server — Browse and open a specification stored on a connected server. This option appears when you are connected and authenticated to a server (see Tutorial 10).
⚡ New users should start with the Sample Project to familiarize themselves with all features before creating their own IDM specifications.

Quick Start

Get up and running in seconds. This tutorial walks you through opening the sample project and arriving at the main BPMN editor workspace.

From startup to the BPMN editor in one click

Steps

1
On the startup screen, click Sample Project.
2
The IDM Header panel opens automatically on the left, showing the specification title: "Information Delivery Manual for the Generic Data Exchange."
3
Close the header panel by clicking the × button (or clicking the Header icon again) to reveal the full BPMN canvas.

You are now in the main workspace. The BPMN diagram shows two swim lanes (Sender and Receiver) with tasks, gateways, and data objects representing a complete exchange process.

📝 The title bar displays the current specification name. The status bar at the bottom shows the number of Exchange Requirements (ERs), Library items, server connection status, and the validation state.

Create a BPMN Diagram

The BPMN editor is the heart of IDMxPPM. It provides a full BPMN 2.0 process diagram canvas where you model the information exchange workflow between participants.

Click BPMN elements to access context actions and assign Exchange Requirements

Key Interactions

1
Add elements from the vertical palette on the left: tasks, events, gateways, data objects, and sequence flows.
2
Click a task to open the Task Detail Panel on the right, where you can edit the task's name and documentation (see Tutorial 12).
3
Double-click a data object (e.g., "er_PrepData") to open the ER Panel, which lets you edit the Exchange Requirement linked to that data object.
4
Use drag-and-drop to reposition elements. Connect them with sequence flows by dragging from one element's edge to another.

The BPMN canvas supports standard BPMN 2.0 elements: start/end events, tasks (User Task, Service Task, etc.), exclusive/parallel gateways, message flows, data objects, and swim lane pools. The toolbar beneath the canvas provides zoom, undo/redo, auto-layout, and export buttons.

📝 Data objects are the bridge between the BPMN process map and Exchange Requirements. Each data object should be assigned to an ER to create a complete IDM specification. Tasks can carry their own name and documentation for process documentation purposes.

Header Information

The IDM Header panel captures all the metadata for your specification. This information appears in exported documents and identifies the IDM uniquely.

Header metadata: title, version, status, language, authors, and revision history

Header Fields

1
IDM Title — The full name of your Information Delivery Manual specification (displayed in the title bar).
2
Version & Status — Track the specification version (e.g., "1.0") and its current status (Draft, Review, Published, Deprecated).
3
Language — Set the specification language (e.g., "EN - English") for multi-language IDM support.
4
Authors — Add one or more authors with name, organization, and role fields. Scroll down to access the full author list.
5
Revision History — Maintain a chronological log of changes with date, author, and description for each revision.
⚡ The header metadata is included in all export formats (idmXML, HTML, ZIP) and is required for ISO 29481-3 compliance.

Use Case & Actors

UPDATED in v1.2.2

The Use Case panel defines the context and purpose of the information exchange. It captures who is involved, what the exchange achieves, and which project phases it applies to. In v1.2.2, actors are now visually linked to BPMN Pools and Lanes with bidirectional synchronization.

Use Case: classification, actor roles (Sender/Receiver), and project phase mapping

Use Case Fields

1
Region & Language — Select the target region and language for the use case. Choose "International" for region-agnostic specifications.
2
Optional fields — Expand to add Purpose, Summary, Classification, Aim & Scope, Benefits, Limitations, Preconditions, and Postconditions.
3
Actor Roles — Define actors and their roles. Each actor has a two-line layout: Name + Badge on Line 1, Role on Line 2. See Tutorial 13 for BPMN linking details.
4
Sub-Actors — Add sub-actors under each actor. Sub-actors correspond to BPMN Lanes within a Pool and can also have roles.
5
Target Project Phases — Check the applicable project stages from ISO 22263, AIA B101, or RIBA Plan of Work.

ISO 22263 AIA B101 (US) RIBA Plan of Work (UK)

Actor Swimlane Badges

Each actor displays a colored badge indicating its BPMN association:

P
Swimlane (Pool) — Green badge. The actor is linked to a BPMN Pool. Renaming the actor updates the Pool name and vice versa.
L
Swimlane (Lane) — Green badge. The sub-actor is linked to a BPMN Lane within a Pool. Names sync bidirectionally.
M
Manual — Gray badge. The actor is not linked to a BPMN element. Click the badge to open the link modal and associate it with a Pool or Lane.
⚡ When you add sub-actors to a Pool-linked actor, BPMN Lanes are automatically created in the Pool. See Tutorial 13 for the full actor-BPMN linking workflow.

Exchange Requirements

UPDATED in v1.2.2

Exchange Requirements (ERs) are the core of the IDM specification. They define what information needs to be exchanged, broken down into hierarchical Information Units with data types, optionality, and external schema mappings. In v1.2.2, the ER hierarchy toolbar now includes Import/Export ER buttons, and Information Units support example images.

ER hierarchy with er_PrepData expanded to show Information Units

Working with ERs

1
Hierarchy View — The left panel shows the ER tree. The root ER is the top-level node containing all sub-ERs and their information units.
2
Toolbar — Use the toolbar to add (+), delete, reorder, expand, or collapse ERs. Import ER and Export ER buttons let you share ERs as erXML files.
3
Individual ER View — Click any ER to open its detail panel on the right. The root ER itself appears as the first row in the tree table and can be selected for editing.
4
Information Units — Each unit has a name, data type, description, required/optional toggle, and support for example images (upload reference images in the detail panel).
5
Detail Panel — The resizable bottom panel provides an editor for the selected item (ER, Sub-ER, or IU). Drag the resize handle to adjust the split.
6
Unsaved Changes — If you have pending edits in the detail panel and click another item, you'll be prompted to apply or discard your changes.
⚡ ERs can be nested to create complex hierarchies. Leaf ERs (those without sub-ERs) must have at least one Information Unit. ERs that only contain sub-ERs are valid per ISO 29481-3.

Validation

Before exporting, validate your IDM specification against ISO 29481-3 requirements. The validation engine checks for completeness, consistency, and compliance automatically. In v1.2.2, validation recursively checks all sub-ERs in the hierarchy.

Validation Results: 0 errors, 0 warnings — ready for export

Steps

1
Click the Validate button (checkmark icon) in the left sidebar.
2
The Validation Results dialog appears, showing the count of Errors, Warnings, and Total issues.
3
If all checks pass, you will see a green checkmark with "All validation checks passed!" and the message "Your IDM specification is ready for export."

The validation engine performs an ISO 29481-3 Compliance Check that verifies:

BPMN structure ER hierarchy (recursive) Data object linkages Required metadata Information Unit integrity Sub-ER validation

📝 If errors are found, each issue is listed with a description and a link to the offending element. Fix the issues and re-validate until all checks pass. You can still save incomplete work even with validation errors.

Save & Export

UPDATED in v1.2.2

IDMxPPM supports multiple export formats to suit different downstream workflows — from full project files to ISO-compliant XML, self-contained HTML documents, and server storage. As of v1.2.2, all idmXML exports use idmXSD v2.0 exclusively with the official ISO namespace.

Choose from multiple export formats, each suited to a different workflow

Steps

1
Click the Save & Export button (download icon) in the left sidebar.
2
Enter a File name (defaults to the Short Title, e.g., "GDE-IDM").
3
Select an export format from the list (see table below).
4
Click Export. In the desktop app, a native OS save dialog lets you choose the save location and confirm overwrites.
Format Extension Description
IDM Project .idm Full project file with all data and library — use this to save your work and reopen later.
idmXML .xml ISO 29481-3 compliant XML (idmXSD 2.0) with official ISO namespace. The standard format for machine-readable IDM exchange.
HTML Document .html Self-contained HTML with embedded images and BPMN diagram (SVG). Ideal for sharing and printing to PDF.
ZIP Bundle .zip Archive containing idmXML + BPMN + images + project data in one package.
BPMN Only .bpmn Process map diagram only — standard BPMN 2.0 XML for use in other BPM tools.
Save to Server cloud Save/update the specification on the connected server. Requires authentication (see Tutorial 10).
XSLT Template .xsl Download the default XSLT stylesheet to customize HTML export formatting.
📝 The v1.0 idmXML export option has been removed in v1.2.2. All exports now use idmXSD v2.0 with the official ISO namespace (standards.iso.org/iso/29481/-3/ed-2/en). Import still supports both v1.0 and v2.0 files.
⚡ Quick-export buttons are also available in the bottom toolbar: SVG | BPMN | PNG for rapid diagram-only exports. When closing a project with unsaved changes, choosing "Save" opens the export dialog and automatically closes the project after saving.

Server Connection

IDMxPPM can optionally connect to a self-hosted server for centralized IDM specification storage and multi-user collaboration. The server is fully optional — the app works 100% offline with local file save/load.

Server Prerequisites

Before connecting, a server must be deployed. The following are required on the machine hosting the server:

◆
Docker Desktop (v4.0+) — download from docker.com. Includes Docker Engine and Docker Compose. Available for Linux, macOS, and Windows.
◆
RAM: 1 GB minimum (2 GB recommended) for MongoDB and the API server.
◆
Disk: 1 GB minimum, plus storage for specifications.
◆
Network: Client machines must be able to reach the server via HTTP/HTTPS (same LAN, VPN, or public URL).
📝 If you prefer not to use Docker, you can install the server manually with Node.js 20+ and MongoDB 7+. See the API User Manual (docs/API_User_Manual.md) for manual deployment instructions.

Connecting to a Server

1
Click the Server icon in the vertical menu bar (cloud icon).
2
Enter the Server URL (e.g., http://localhost:3001) and click Connect.
3
Login with your email and password, or Register a new account (the first user automatically becomes admin).
4
Once authenticated, a green dot appears on the server icon and the status bar shows "Server".

The server connection modal has three states:

Not Connected Connected (login/register) Authenticated

User Roles

A
Admin — Full access to all specs. Can update and delete any specification. The first registered user becomes admin.
E
Editor — Can create, update, and delete own specs. Cannot modify others' specifications.
V
Viewer — Can browse and open any spec. Can create and manage own specs only.
⚡ With Docker Desktop installed, server deployment is as simple as: cd server && cp .env.example .env && docker-compose up -d. See the API User Manual for details.

Server Browser

The Server Browser lets you browse, search, and open IDM specifications stored on the connected server. It provides a full-featured table with search, filtering, sorting, and pagination.

Opening the Server Browser

1
Click "Open from Server" on the Startup Screen (requires server connection), or use the server connection modal.
2
The Server Browser modal opens, showing all available specifications on the server.

Browsing & Searching

1
Search — Type in the search box to filter by title or short title (full-text search).
2
Filter by Status — Use the dropdown to show only specs with a specific status (NP, WD, CD, DIS, IS).
3
Sort — Click any column header to sort by Title, Status, Version, ER Count, Author, or Modified Date.
4
Paginate — 15 specs per page with Previous/Next navigation.

Status badges are color-coded: NP WD CD DIS IS

⚡ After opening a spec from the server, use "Save to Server" in the export dialog to push your changes back. The status bar shows "Server (synced)" to confirm.

Task Detail Panel

NEW in v1.2.2

The Task Detail Panel allows you to edit the name and documentation (description) of any BPMN task directly from the UI. Task properties are stored as standard BPMN 2.0 elements and are automatically preserved through all save and export operations.

🗒 Click any BPMN task to open the Task Detail Panel on the right

Opening the Panel

1
Click any task on the BPMN canvas (Task, User Task, Service Task, Send Task, Receive Task, Manual Task, Business Rule Task, Script Task, Call Activity, or Sub Process).
2
The Task Detail Panel opens on the right side, showing the task type (e.g., "UserTask") and editable fields for Name and Documentation.
3
Edit the Name (updates the label on the BPMN canvas) and Documentation (a longer description of the task's purpose).

Saving Changes

1
Click the Save button, or use the keyboard shortcut Ctrl/Cmd + Enter.
2
An "Unsaved" badge appears when you have pending changes. The Save button is disabled when there are no changes.
3
Click the × button or click on an empty area of the canvas to close the panel.
📝 The Task Detail Panel and the ER Panel are mutually exclusive — opening one automatically closes the other. Click a task to see its details; double-click a data object to see its ER.
⚡ Task name and documentation are standard BPMN 2.0 properties. They are preserved in all export formats (.idm, .xml, .zip, .bpmn, server) without any extra configuration.

Actor-to-BPMN Linking

NEW in v1.2.2

Actors in the Use Case panel can be linked to BPMN Pools and Lanes, creating a bidirectional relationship between the process map and the use case specification. When linked, names stay synchronized — renaming an actor updates the Pool/Lane on the canvas, and vice versa.

🔗 Link actors to BPMN Pools and Lanes with visual badges and automatic synchronization

Linking an Actor to a Pool

1
Open the Use Case panel from the sidebar. Each actor shows a colored badge indicating its BPMN association.
2
Click the Manual badge (gray) on an unlinked actor. A Link Modal appears listing all available BPMN Pools and Lanes.
3
Click on an available Pool to establish the link. The badge changes to Swimlane (Pool) (green).
4
The actor name and Pool name are now synchronized. Editing either one updates the other automatically.

Automatic Lane Creation

1
With a Pool-linked actor selected, click + Sub-Actor to add a sub-actor.
2
A BPMN Lane is automatically created inside the linked Pool for each sub-actor.
3
Sub-actor names are synced with Lane names. Each sub-actor can also have a Role field.

Merging Actors

1
If you link an actor to a Pool that is already linked to another actor, a merge confirmation dialog appears.
2
Merge — The source actor is removed and its data (role, sub-actors) is merged into the existing actor linked to that Pool.
3
Cancel — The link operation is cancelled and both actors remain unchanged.

Bidirectional Sync

⇄
Rename Actor in Use Case panel → Pool/Lane name updates on BPMN canvas.
⇄
Rename Pool/Lane on BPMN canvas → Actor/sub-actor name updates in Use Case panel.
⇄
Delete Pool from BPMN → Linked actor is removed (with confirmation dialog).
⇄
Delete Lane from BPMN → Linked sub-actor is removed from the actor's list.
📝 Sub-actor roles are exported to idmXML as <subActor><classification name="role"/></subActor> and properly restored on import.
⚡ When adding actors manually (via the + Add Actor button), a BPMN Pool is automatically created if a project is open. You can also create Pools on the BPMN canvas first — they will appear as actors in the Use Case panel.

Importing Legacy Files

NEW in v1.2.2

IDMxPPM supports importing from multiple file formats, including full support for legacy xPPM files. In v1.2.2, the xPPM importer can automatically load external BPMN diagrams and images from adjacent folders, and the idmXML importer can resolve external file references from v1.0 files.

📂 Import from .idm, .xml, .zip, .xppm, .bpmn, and .erxml files

Supported Import Formats

⚫
.idm — Full IDMxPPM project file with BPMN diagram, ERs, header, and use case data.
⚫
.xml — idmXML (ISO 29481-3). Supports both v1.0 and v2.0 with automatic version detection. Embedded BPMN and images are restored.
⚫
.zip — ZIP bundle containing idmXML, BPMN, images, and project data.
⚫
.xppm — Legacy xPPM format. Converts header, use case, actors with roles, ERs with description figures, and IU example images.
⚫
.bpmn — BPMN 2.0 diagram only (no ER or header data).

xPPM Import (Electron Desktop App)

1
Click Open Project and select a .xppm file.
2
The BPMN diagram is automatically loaded from the Diagram/ folder adjacent to the .xppm file.
3
All referenced images are automatically loaded from the Image/ folder and converted to embedded base64.
4
If the xPPM file contains multiple top-level ERs, a Root ER Selection dialog lets you choose or create a root ER.

idmXML v1.0 External References

1
When importing a v1.0 idmXML file that references external files (BPMN diagrams and images) instead of embedding them, the app attempts to load them automatically.
2
The app tries multiple candidate paths: the original path, ./Diagram/ subfolder, and ./Image/ subfolder.
3
If some files cannot be found, the import continues with placeholder references. Missing images can be re-added manually.
📝 Full xPPM and external reference resolution requires the Electron desktop app. In the browser version, external files (BPMN diagrams and images stored as separate files) cannot be loaded automatically.
⚡ When importing files with UTF-8 BOM (byte order mark), the app automatically strips the BOM to prevent XML parsing errors. This is common with files saved by some Windows text editors.
